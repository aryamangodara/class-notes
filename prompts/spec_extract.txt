You are extracting a structured curriculum map from an official exam-board
specification. From the supplied spec pages covering ONE topic, produce a
TopicSpec.

Extract:
- board, subject, level, unit, topic
- prerequisites the spec assumes
- the exact learning objectives: verbatim codes and statements, with `tier`
  (e.g. Core / Supplement) and `command_words` where the spec marks them
- depth_profile: how far THIS board takes the topic, including anything the spec
  explicitly excludes
- assessment_notes: how the topic is examined (paper, question styles, command
  words, mark-scheme expectations)
- source: the document and page range you extracted from

Rules:
- Do NOT invent objectives that are not present in the spec.
- Quote objective codes exactly as printed.
- If the spec marks Core vs Supplement (or AS vs A-Level), preserve that in `tier`.

Return structured JSON matching the TopicSpec schema.
